## 技巧
1. 哑节点：方便指针少移动一个节点；
2. 删除节点：需要有pre，pre.next = pre.next.next
3. 寻找规律：简单情形模拟、画图、发现规律
4. 分类讨论法：如果情形分为多级讨论，每次分类时都穷尽所有情况，如二分类，那么就可以穷尽所有情形
5. set: add()添加失败返回false，用于替换contain判断；只需判断是否存在不在乎对应值时，用来替换map；
